Modern Java web application with Spring

이런 기술의 홍수속에서 눈여겨볼만한 웹기술과 이와 관련된 스프링의 웹기술을 살펴볼 수 있는 자리를 마련했습니다.

Spring 4.x Web Application / 김지헌

2015년 7월 31일 Spring Framework 4.2가 출시되었습니다.

CORS(Cross-origin resource sharing), Server-Sent Events 지원과 비동기 요청 처리를 위한 기능 확장, RestTemplate의 OkHttp 통합, 그리고 자바 8의 새 자바스크립트 엔지 나즈혼(Nashorn)을 활용한 ScriptTemplateView까지 최신 웹 경향을 반영한 다양한 기능들이 추가되었습니다.

이 세션에서는 Spring 4.x 버전의 웹 기능들을 살펴보는 자리를 만들고자 합니다.

스크립트 없는 자바 웹 애플리케이션 만들기 / 이재일

웹 애플리케이션은 크게 클라이언트에서 동작하는 프론트엔드(Front-end)와 서버 측에서 동작하는 백엔드(Back-end)로 나눌 수 있습니다.
프론트엔드는 HTML, CSS, JavaScript를 사용해서 개발하고, 백엔드는 자바, Python, Ruby, Scala 등으로 개발합니다.

웹 애플리케이션을 개발하려면 백엔드 기술뿐만 아니라 프론트엔드 기술도 능숙하게 다루어야 합니다. 하지만 두 영역을 함께 다루려면 비용과 노력이 많이 들고, 특히 백엔드에 비해 프론트엔드는 컴포넌트나 모듈화의 개념이 약합니다. 따라서 효율적으로 개발하기가 쉽지 않습니다.
Vaddin은 자바만 사용해서 백엔드와 프론트엔드를 한 번에 개발할 수 있는 오픈소스 웹 애플리케이션 프레임워크입니다. 이 기술을 사용하면 프론트엔드 기술을 몰라도 Java 언어만으로 손쉽게 좋은 품질과 기능을 갖춘 RIA(Rich Internet Application) 웹 애플리케이션을 만들 수 있습니다.

이 세션은 Vaddin을 소개하고, Vaddin을 사용해 하나의 웹 애플리케이션을 직접 개발하는 과정을 다룹니다.
